# UI設計

## 画面一覧

- ログイン画面
- グループ一覧画面
- グループ未所属画面
- グループ作成画面
- 招待確認画面
- ToDo一覧画面

---

## 画面遷移

```
ログイン
　↓
グループ一覧 →（未所属の場合）グループ未所属画面
　↓
グループ選択 → ToDo一覧

グループ作成 → グループ一覧へ
招待URLアクセス → 招待確認画面 → ToDo一覧
```

---

## 各画面詳細

### ログイン画面

- ID入力
- パスワード入力
- ログインボタン
- エラー表示：「ログイン情報が違います」

### グループ一覧画面

- グループ名一覧
- メンバー数表示
- グループ作成ボタン
- 招待URL確認機能（グループメンバー全員に表示）

### グループ未所属画面

- 「グループがありません」と表示
- グループ作成ボタン

### グループ作成画面（モーダル）

- グループ名入力
- 作成ボタン
- 作成後に招待URL表示

### 招待確認画面

- グループ名表示
- 作成者表示
- 参加ボタン

---

## ToDo一覧画面

> ※対象グループに所属しているユーザーのみ閲覧・操作可能

### 表示項目

- タイトル
- priority（低・中・高）
- ステータス（未完了・完了）
- 作成者
- 期日

### 並び順

- priority降順（高 → 低）
- 同priority内で期日昇順

### 機能

- 未完了フィルター
- ToDo追加ボタン
- 編集ボタン → 作成者のみ表示
- 削除ボタン → 作成者のみ表示

---

## ToDo操作

### 作成（モーダル）

- タイトル
- description
- 期日
- priority
- 保存ボタン

### 編集（モーダル）

- 作成と同様

### 削除

- 確認ダイアログ表示
- OKで削除

**削除成功時**

- 一覧から削除
- 「削除しました」と表示（トースト）

**削除失敗時**

- 「削除権限がありません」と表示

---

## 非同期処理（Knockout.js）

### ToDo

- 追加 → 即一覧に反映
- ステータス変更 → 即更新
- 削除 → 即一覧から削除

### UX

- 非同期通信中はローディング表示
- エラー時はメッセージ表示

---

## 権限制御

### 前提条件

- 対象ToDoのグループに所属しているユーザーのみ操作可能

### ToDo削除権限

- 作成者のみ削除可能

### ToDo編集権限

- 作成者のみ可能

---

## バリデーション

- タイトル：必須
- グループ名：必須
- ID / パスワード：必須
